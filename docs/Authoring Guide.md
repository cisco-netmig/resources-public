# Netmig Script Authoring Guide

This guide outlines how to structure, develop, and register your custom Python scripts for integration with the Netmig tool. It covers the directory structure, file templates, GUI design principles using Qt Designer, and the usage of the Add Script dialog within Netmig.

---

## 1. Authoring Workflow Overview

If you're currently running your script from the command line, follow these steps to integrate it into Netmig:

### Step 1: Identify Components

- **Inputs**: What parameters does your script need?
- **Outputs**: What results should the user see?
- **Actions**: What processes are triggered (e.g., SSH login, file transfer, data parsing)?

### Step 2: Design the GUI with Qt Designer

1. Launch Qt Designer → Create a **QWidget Form**.
2. Add input fields, buttons, labels, etc., for required parameters and results.
3. Save the form as `yourform.ui`.
4. Convert the `.ui` file to Python:
   ```
   pyuic5 yourform.ui -o ui.py
   ```
5. Create a `Form` class in `ui.py` to encapsulate behavior.

### Step 3: Add Background Logic

Place your script logic in `workers.py`, using:

- `QThread` or `QRunnable` for non-blocking background execution.
- Signals and slots to communicate with the GUI.

### Step 4: Use the Netmig "Add Script" Dialog

Once the GUI and logic are ready:

- Go to **File > Add Script** in Netmig.
- Provide:
  - Script Name
  - Author, Version
  - Icon (`.ico`, optional)
  - README file (optional)

Netmig will:

- Rename `main.py` → `__main__.py`
- Store metadata in `__meta__`
- Rename `icon` → `__icon__.ico`
- Rename `README` → `README.md`
- Ensure the folder is Netmig-compatible

## 2. Script Folder Structure

### Initial Author Folder (Before Integration)

```plaintext
your_script/
├── __init__.py         # Mandatory: declares the Form class
├── main.py             # Mandatory: GUI entry point
├── ui.py               # Recommended: UI class based on Qt Designer
├── workers.py          # Recommended: background logic
```

### After Using Netmig’s Add Script
```plaintext
your_script/
├── __init__.py
├── __main__.py         # Renamed from main.py
├── __meta__            # Metadata auto-generated by Netmig
├── __icon__.ico        # Optional icon if provided
├── README.md           # Optional readme if provided
├── ui.py
├── workers.py
└── assets/             # Optional: templates, configs, static files
```

---

## 3. Standard File Templates

| Filename      | Required | Purpose                                      |
|---------------|----------|----------------------------------------------|
| `__init__.py` | ✅ Yes   | Declares the `Form` class for embedding in Netmig |
| `main.py`     | ✅ Yes   | Standalone entry; renamed to `__main__.py`   |
| `ui.py`       | ⚠️ Recommended | Contains GUI design & behavior logic        |
| `workers.py`  | ⚠️ Recommended | Contains background jobs or network logic  |
| `icon.ico`    | Optional | Script icon (renamed to `__icon__.ico`)      |
| `README.md`   | Optional | Script documentation                         |
| `assets/`     | Optional | Contains templates, static files, etc.       |

## 4. Recommended Template Snippets

### `__init__.py`
```python
from .ui import Form
```

### `main.py`
```python
from ui import Form
from PyQt5.QtWidgets import QApplication
import sys

if __name__ == "__main__":
    app = QApplication(sys.argv)
    window = Form()
    window.show()
    sys.exit(app.exec_())
```

### `ui.py`
```python
from PyQt5.QtWidgets import QWidget

class Ui_Form:
    def setupUi(form):
        # Generated UI code from Qt Designer
        pass

class Form(QWidget, Ui_Form):
    def __init__(self):
        super().__init__()
        self.setupUi(self)
        # Connect signals, buttons, and worker threads
```

### `workers.py`
```python
from PyQt5.QtCore import QObject, pyqtSignal, QRunnable, QThreadPool

class ExampleWorker(QRunnable):
    def run(self):
        # Background job logic here
        pass
```
